# 生成最高权限管理员账号（SND 小程序）

登录页原本「所属单位 / 机构·班组」为空，导致无法注册。根因是 `orgs` 集合为空且无管理员。
本项目已内置「一键生成管理员」能力，也可在云控制台直接插库。

> 角色与权限模型（最新）：
> - `admin`（**小程序管理员**）：拥有小程序**全部数据管理权限**的唯一最高角色。
>   可进入「系统管理」页，进行组织与用户的增删改；**不开放自助注册**，必须由系统初始化或控制台分配。
> - `lead`（专班负责人）：`hasRole()` 对任意角色都返回 true，绝对最高；可进入系统管理。
> - `supervisor`（安监部管理人员）：`requireAdmin()` 放行，可进入系统管理。
> - `project_lead`（项目部负责人）/ `worker` / `group_lead` / `safety_officer` / `lease_admin`：
>   均**支持自助注册与登录**，但无系统管理权限。
>
> 即：专班负责人、项目部负责人、安监部管理人员都能手动注册并登录；
> 除此之外的「小程序管理员（admin）」是唯一拥有全部数据管理权限的角色。

---

## 方法一：登录页内置按钮（最省事，推荐）

1. 部署 `auth`、`system` 两个云函数并确认 `users` / `orgs` 集合已存在。
2. 微信开发者工具中预览/真机打开小程序，进入登录页。
3. 点击底部 **「⚙ 初始化管理员账号（仅首次）」**。
4. 确认后，当前微信身份会被设为 **Jousts / qwer1234（小程序管理员 admin）**。
5. 切到「登录」标签，输入 `Jousts` / `qwer1234` 即可登录。
6. 进入「我的 → 系统管理」即可管理组织架构与用户。

> 幂等保护：一旦已存在 `admin` / `lead` / `supervisor`，按钮会提示「管理员已存在」，不会覆盖。

---

## 方法二：云开发控制台直接插库（无需打开小程序）

1. 微信开发者工具 → 云开发 → 数据库 → 集合 `users` → 新建记录（JSON 模式）。
2. 粘贴以下内容（密码已是哈希，切勿写明文）：

```json
{
  "username": "Jousts",
  "nickname": "Jousts",
  "password": "472f0c3b35965d7c84389d53aea8f9f3a6a055c2",
  "role": "admin",
  "bound": true,
  "status": "active"
}
```

3. 保存。该账号 `openid` 为空，首次用 `Jousts / qwer1234` 登录时会自动绑定当前微信身份。
4. 若想用绝对最高权限（专班负责人），把 `"role": "admin"` 改成 `"role": "lead"` 即可；
   若只需安监部管理权限，改为 `"role": "supervisor"`。

> 密码哈希算法固定为：`sha1('tms_' + 明文密码)`。改密码时必须插哈希值，例如
> `qwer1234` 的哈希就是上面的 `472f0c3b35965d7c84389d53aea8f9f3a6a055c2`。
>
> ⚠️ 控制台「新建记录（JSON 模式）」的 `tcb/PutItem` 入参校验很严格，以下写法都会触发
> `InvalidParameter / Check request parameter fail`：
> - **空字符串字段**：`"openid": ""`、`"unitId": ""`、`"orgId": ""` → 报「key 或 value 不能为空」。
> - **`$` 开头的键 / MongoDB 扩展 JSON**：`"createdAt": { "$date": "..." }` 这类写法控制台不会解析，
>   `$date` 键以 `$` 开头会被拒 → 报「Check request parameter fail」。**本示例已省略这些字段。**
> - 字段名不要含 `.`，也不要以 `_` / `$` 开头（除 `_id` 外）。
> 日期如需写入，请用控制台可视化编辑器的日期选择器；本账号首登后由系统自动补全 `openid`。

## 方法三：云函数测试调用（无界面）

1. 云开发 → 云函数 `system` → 测试 / 日志 → 调用（**需先部署最新 `system` 云函数**）。
2. 请求体：

```json
{ "action": "seedAdmin", "payload": { "username": "Jousts", "password": "qwer1234" } }
```

3. 返回 `{ "code": 0, "data": { "username": "Jousts", "role": "admin" } }` 即成功。
   （内部用 `getWXContext().OPENID` 建档；首次登录时自动绑定真实微信身份。）

## 之后如何修复「所属单位 / 机构·班组 为空」

管理员登录后：
- 若组织树为空：系统管理页点 **「恢复默认」** 写入示例结构
  （单位：平台 / 安装公司 / 广安公司 / 分包1 / 分包2；安装公司下设工程部、调试班；
  分包1 下设工程部，其下再设木工班、电工班）。
- 或手动 **新增组织 / 编辑 / 删除** 任意 所属单位、项目部、机构·班组。
- 注册页的组织树来自 `system` 云函数 `orgTree`，会自动读取上述数据。
